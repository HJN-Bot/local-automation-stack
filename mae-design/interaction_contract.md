# MAE Layer 1 — 交互层协议（Interaction Contract）

> Andrew（SAM角色）作为Orchestrator，接收用户需求后自动完成全流程

---

## 交互流程

```
用户提出需求
    ↓
┌────────────────────────────────────────────────┐
│  Phase 1: AgentPlan 设计                        │
│  ├─ 分析需求复杂度 → Tier判定（T0/T1/T2/T3）    │
│  ├─ 选择需要的Agent角色                         │
│  ├─ 定义每个Agent的：                           │
│  │   ├─ 职责（做什么）                          │
│  │   ├─ 输入（需要什么）                        │
│  │   ├─ 输出（产出什么）                        │
│  │   ├─ 工具（用什么能力）                      │
│  │   └─ 验收标准（怎么算完成）                  │
│  ├─ 定义信息回流路径                            │
│  └─ 定义状态机协作规则                          │
│                                                 │
│  输出：AgentPlan（一页纸）→ 用户确认             │
└────────────────────┬───────────────────────────┘
                     ↓ 用户确认
┌────────────────────────────────────────────────┐
│  Phase 2: 自动编排执行                           │
│  ├─ 创建Airtable任务（Status=LOADED）           │
│  ├─ spawn sub-agents（通过sessions_spawn）      │
│  ├─ 分派任务给各Agent                           │
│  ├─ 监控执行状态                                │
│  └─ 汇总中间产物                                │
└────────────────────┬───────────────────────────┘
                     ↓
┌────────────────────────────────────────────────┐
│  Phase 3: 评估与自适应                           │
│  ├─ 检查每个Agent的输出质量                     │
│  ├─ 如果BLOCKED → 分析原因 → 重设计该节点       │
│  ├─ 如果质量不达标 → 反馈修正 → 重新执行        │
│  └─ 如果全部通过 → 汇总最终产物                 │
└────────────────────┬───────────────────────────┘
                     ↓
┌────────────────────────────────────────────────┐
│  Phase 4: 交付与验收                             │
│  ├─ 产出EvidencePack（证据包）                  │
│  ├─ 写回Dashboard/Airtable                     │
│  ├─ 通知用户验收                                │
│  └─ 用户确认 → DONE / 用户拒绝 → 回Phase 3     │
└────────────────────────────────────────────────┘
```

## Tier 判定规则

| Tier | 复杂度 | Agent数 | 示例 |
|------|--------|---------|------|
| T0 | 单步直接执行 | 1 (SAM自己) | 查天气、简单问答 |
| T1 | 需要工具 | 1-2 | 抓取单个URL并总结 |
| T2 | 多步编排 | 2-3 | Follow Builder（抓取→总结→写回） |
| T3 | 复杂多Agent | 3-4+ | 视频生产线（脚本→TTS→字幕→渲染→发布） |

## SAM（Andrew）的自动编排行为

### 收到需求后：
1. **Tier判定**：分析复杂度，决定用几个Agent
2. **出AgentPlan**：一页纸展示给用户
3. **等用户确认**：用户说"执行"后开始

### 执行中：
4. **按flow顺序spawn Agent**：通过`sessions_spawn`
5. **传递context**：前一个Agent的输出 = 下一个Agent的输入
6. **监控状态**：检查每个Agent是否完成

### 遇到卡点时：
7. **分析原因**：工具不可用？权限不够？内容质量差？
8. **重设计**：修改该节点的方案（换工具/降级/人工介入）
9. **继续执行**：从卡点处恢复

### 完成时：
10. **汇总EvidencePack**：收集所有产物和链接
11. **通知用户验收**：展示结果
12. **写回系统**：更新Airtable/Dashboard状态
