# WikiBot Writing Spec

## Goal
Make Wiki writing feel more natural, grounded, and useful — not like generic AI templates.

## Writing principles
1. Write from the actual task / conversation context, not from abstract templates.
2. Preserve the reasoning path: what happened, what mattered, what was decided.
3. Keep some human texture where useful: concise framing, original angle, occasional key quoted line.
4. Prefer readable structure over rigid report formatting.
5. Output must work well in Feishu Wiki and remain understandable when mirrored to Markdown.

## Recommended page structure
Use as default skeleton, not as a rigid template:
- Context
- What happened
- Key judgment / takeaway
- Deliverable / output
- Evidence / links
- Next step

## Naturalness rules
- Do not make every page sound like a formal report.
- Allow concise work-note tone when appropriate.
- Use headings only when they improve readability.
- Do not overuse bullets if short paragraphs are more natural.
- Avoid filler phrases like “Based on the discussion above”.
- Prefer specific statements over abstract generic summary language.

## What to avoid
- generic summary templates
- detached narration with no user/task context
- long ceremonial intros
- repetitive section headers that add no value
- pretending certainty when context is incomplete

## Feishu-specific quality goal
The page should feel like something Jianan would actually want to re-open later:
- clear enough to scan quickly
- specific enough to be actionable
- human enough to preserve intent
